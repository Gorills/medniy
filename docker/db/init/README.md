# Инициализация MySQL

Файлы `.sql`, `.sql.gz` и исполняемые `.sh` в этом каталоге выполняются **один раз** при первом создании пустого тома `db_data`.

## Снимок по умолчанию

`01_medniy_seed.sql.gz` — полный дамп приложения (схема + данные, включая `migrations`). После первого `docker compose up` отдельный `make migrate` обычно ничего не добавляет (все миграции уже в таблице `migrations`).

## Повторная загрузка с нуля

```bash
docker compose down
docker volume rm medniy_db_data
docker compose up -d
```

Все данные в MySQL будут удалены; при следующем старте тома init выполнится снова.

## Свой дамп

Замените или добавьте `.sql`/`.sql.gz` (имя с префиксом, например `02_custom.sql`), затем пересоздайте том, как выше. Большие дампы в git не обязательны — можно класть только на сервер перед первым запуском.
